我的封装之路(一)

发表于 2017-01-09 |

前言

做Android开发已经有一段时间了，从开始的什么不懂到现在的大概能懂。从开始的一遍遍认真敲代码到现在的适时偷懒，学了那么久了自己也有点感触或者想法，那就是不能老是一次次的敲着重复低级的代码，这是在我使用过一些别人封装的库或者框架之后得到的体会，特别是一些好的库或者框架，很多都是两三行代码就搞定了，now，我也要开始严格要求自己尝试去封装一些东西。

既然有了这个决心那就开始吧，当然我会根据具体的情况来实现任务进度，比如这次就先封装一个RecyclerView的Adapter和ViewHolder。

ViewHolder封装 AbsAdapter

RecyclerView的标准配置ViewHolder、Adapter，ViewHolder是为了优化性能而提出的一个类，不只是RecyclerView，之前的ListView也是建议使用ViewHolder来配合。然后每次使用RecyclerView都要重复的实现这两个类，这多少让人有点不耐烦。下面就让我对这些重复的操作进行一下封装。首先是ViewHolder

public class BaseViewHolder extends RecyclerView.ViewHolder {

  private SparseArray<View> mViews;

  public BaseViewHolder(View view){
    super(view);
    mViews = new SparseArray<>();
  }

  public BaseViewHolder(ViewGroup parent, int resId) {
    super(LayoutInflater.from(parent.getContext()).inflate(resId, parent, false));
    mViews = new SparseArray<>();
  }

  public <T extends View> T getView(int viewId) {
    View view = mViews.get(viewId);
    if (view == null) {
      view = itemView.findViewById(viewId);
      mViews.put(viewId, view);
    }
    return (T) view;
  }

  /*=========================事件==========================*/
  public BaseViewHolder setOnClickListener(int viewId, View.OnClickListener listener) {

    View view = mViews.get(viewId);
    view.setOnClickListener(listener);
    return this;
  }
}

创建一个BaseViewHolder类继承RecyclerView.Adapter,然后使用SparseArray来存储Item的View。重点是getView这个方法，该方法返回的是一个继承View的泛型，通过该方法可以获取到对应的View对象，获取对象之前会进行一些判断操作，首先查看该View是否存储再SparseArray里，是则直接返回该对象，否则创建该对象并保存在SparseArray,使用此方法可以降低性能开销。setOnClickListener为了给View对象设置点击监听的一个方法。以上就是封装的ViewHolder。

Adapter的封装

接下来是对Adapter的封装，对Adapter的封装会比较复杂，使用了两个类来进行封装 。首先是用了一个抽象类AbsAdapter来继承RecyclerView.Adapter主要实现RecyclerView.Adapter中的onCreateViewHolder和onBindViewHolder，这两个方法分别是创建ViewHolder和对ViewHolder进行数据绑定。

@Override public final BaseViewHolder onCreateViewHolder(ViewGroup parent, int viewType) {
  if (viewType == VIEW_TYPE_HEADER) {
    return new BaseViewHolder(mHeaderView);
  } else if (viewType == VIEW_TYPE_FOOTER) {
    return new BaseViewHolder(mFooterView);
  } else {
    return createCustomViewHolder(parent, viewType);
  }
}

@Override public void onBindViewHolder(BaseViewHolder holder, int position) {
  Object value = null;
  switch (holder.getItemViewType()) {
    case VIEW_TYPE_HEADER:
    case VIEW_TYPE_FOOTER:
      break;
    default:
      bindCustomViewHolder((VH) holder, (M) value, position);
      break;
  }
}

其中createCustomViewHolder和bindCustomViewHolder是两个抽象方法，在下一个类中进一步实现。当然在该类还添加了HeaderView和FooterView的一些方法，比如添加或者移除HeaderView、FooterView,总的来说该类并没有对数据进行操作，只是操作了一些View的东西。以下是一些扩展的方法，方法名都比较易懂，就不解释用来干嘛了…

/**
 * 添加HeaderView
 */
public void addHeaderView(View headerView) {
  if (headerView == null) {
    Log.w(TAG, " add the headView is null");
    return;
  }
  this.mHeaderView = headerView;
  notifyDataSetChanged();
}

public void addFooterView(View footerView) {
  if (footerView == null) {
    Log.w(TAG, " add the footView is null");
    return;
  }
  this.mFooterView = footerView;
  notifyDataSetChanged();
}

public void removeHeaderView() {
  if (mHeaderView != null) {
    mHeaderView = null;
    notifyDataSetChanged();
  }
}

public void removeFooterView() {
  if (mFooterView != null) {
    mFooterView = null;
    notifyDataSetChanged();
  }
}

对数据的处理 BaseAdapter

在这个Adapter里面使用泛型的特点进行数据的处理和数据的绑定，首先是要有一个接收数据的对象，这里一般用List对象来接收，在强调一遍，在该BaseAdapter里面主要是对数据的操作，所以这个类主要的功能就是数据的正删改查，这四个点可以扩展出比较多的方法，在这次封装中可能还不够完善，反正是第一次，难免做得不足，所以不足的地方估计要等到以后才能完善了。

  • 构造方法有两个，没什么好说的应该都能看得懂

    public BaseAdapter(Context context) {
  super(context);
  this.mDataList = new ArrayList<>();
}

public BaseAdapter(Context context, List<M> dataList) {
  super(context);
  this.mDataList = new ArrayList<>();
  mDataList.addAll(dataList);
}

  • 第一个关键点就是这里重写了 onBindViewHolder方法

    @Override public void onBindViewHolder(BaseViewHolder holder, int position) {
  super.onBindViewHolder(holder, position);
  Object value = mDataList.get(position);
  if (value != null) {
    switch (holder.getItemViewType()) {
      case VIEW_TYPE_HEADER:
      case VIEW_TYPE_FOOTER:
        break;
      default:
        bindCustomViewHolder((VH) holder, (M) value, position);
        break;
    }
  }
}

    然后实现了这个 bindCustomViewHolder((VH) holder, (M) value, position) 抽象方法转换为具体的数据类型。以下就是该方法的实现

    @Override protected void bindCustomViewHolder(VH holder, M value, int position) {
  if (value != null) {
    bindTo(holder, value, position);
  }
}

    bindTo(holder, value, position) 是一个抽象方法，会被继承至该类的子类所实现。

      /**
     * 重写该方法进行数据绑定
     * value 是集体的数据类型
     */
protected abstract void bindTo(VH holder, M value, int position);

    到此为止就完成了数据的接收和绑定，接下来就是一些扩展方法。

  • 数据的增删改查，注释基本都写上了，而且方法浅显易懂，也就不在多解释了。

      /**
 * 填充数据(会先清除原来的数据)
 *
 * @return true:成功并刷新数据
 */
public boolean addAll(List<M> dataList) {
  this.mDataList.clear();
  boolean result = mDataList.addAll(dataList);
  if (result) {
    notifyDataSetChanged();
  }
  return result;
}

/**
 * 添加单条数据
 *
 * @param data 要添加的数据对象
 * @return true:添加成功并刷新界面
 */
public boolean appendItem(M data) {
  boolean result = mDataList.add(data);
  if (result) {

    if (getHeaderExtraViewCount() == 0) {
      notifyItemChanged(mDataList.size() - 1);
    } else {
      notifyItemInserted(mDataList.size());
    }
  }
  return result;
}

/**
 * 追加数据集合
 *
 * @param dataList 要追加的数据集合
 * @return true:成功追加刷新界面
 */
public boolean appendList(List<M> dataList) {
  boolean result = mDataList.addAll(dataList);
  if (result) {
    notifyDataSetChanged();
  }
  return result;
}

/**
 * 将数据前置
 */
public void proposeItem(M date) {
  mDataList.add(0, date);
  if (getHeaderExtraViewCount() == 0) {
    notifyItemChanged(0);
  } else {
    notifyItemChanged(getHeaderExtraViewCount());
  }
}

/**
 * 将dataList数据前置
 */
public boolean proposeList(List<M> dataList) {
  boolean result = mDataList.addAll(0, dataList);
  if (result) {
    notifyDataSetChanged();
  }
  return result;
}

/**
 * 根据位置获取该数据
 *
 * @return 数据
 */
public M getItem(int position) {
  if (mHeaderView != null && position == 0 || mFooterView != null && position >= mDataList.size
      () + getHeaderExtraViewCount()) {
    return null;
  }
  return mHeaderView == null ? mDataList.get(position) : mDataList.get(position - 1);
}

/**
 * 根据ViewHolder获取该数据
 *
 * @return 数据
 */
public M getItem(VH holder) {
  return getItem(holder.getAdapterPosition());
}
/**
 * 移除一条数据
 */
public void removeItem(int position) {
  if (mHeaderView == null) {
    mDataList.remove(position);
  } else {
    mDataList.remove(position - 1);
  }
  notifyItemRemoved(position);
}

public void removeItem(M data) {
  int index = mDataList.indexOf(data);
  if (index < 0) {
    return;
  }
  mDataList.remove(data);
  if (mHeaderView == null) {
    notifyItemRemoved(index);
  } else {
    notifyItemRemoved(index + 1);
  }
}
/**
 * 更新一个Item
 */
public void updateItem(M date) {
  int index = mDataList.indexOf(date);
  if (index < 0) {
    return;
  }
  mDataList.set(index, date);
  if (mHeaderView == null) {
    notifyItemChanged(index);
  } else {
    notifyItemChanged(index + 1);
  }
}

基本上就差不多这些内容。如果之后更改了，会做进一步的补充。
